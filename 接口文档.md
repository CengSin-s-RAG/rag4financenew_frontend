## API 接口文档

> 后端服务地址：`http://localhost:8086`

---

### 1. 创建新会话（New Session）

请求方式：POST
接口路由：`http://localhost:8086/v2/session/new`

接口参数：无

接口响应：
```json
{
  "session_id": "b199d322-7079-4a1b-958c-1402012d75d1"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| session_id | string | 新会话的 UUID，后续聊天请求需携带此值 |

接口说明：
1. 每次新建对话时调用此接口获取 `session_id`
2. 获取到的 `session_id` 需在后续 `/v2/chat` 请求中携带

---

### 2. 智能对话（RAG Chat）

请求方式：POST
接口路由：`http://localhost:8086/v2/chat`

接口参数：
```json
{
  "question": "白银的蛙跳是什么意思？",
  "session_id": "b199d322-7079-4a1b-958c-1402012d75d1"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question | string | 是 | 用户提问内容 |
| session_id | string | 否 | 会话ID，不传时后端自动生成 UUID |

接口响应：
```json
{
  "session_id": "b199d322-7079-4a1b-958c-1402012d75d1",
  "reply_message": "## 一、概念解释\n\n白银的蛙跳是指..."
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| session_id | string | 本次会话ID，后续请求需携带此值 |
| reply_message | string | AI 回复内容，Markdown 格式 |

接口说明：
1. 建议先通过 `/v2/session/new` 获取 `session_id`，再调用此接口
2. `reply_message` 为 Markdown 格式
3. 后端使用 ReAct Agent，会自动调用 `search_financial_knowledge` 工具检索金融文章

---

### 3. 文档重排序（Rerank）

请求方式：POST
接口路由：`http://localhost:8086/v2/rerank`

接口参数：
```json
{
  "question": "白银的蛙跳是什么意思？",
  "documents": ["文档1内容...", "文档2内容...", "文档3内容..."]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| question | string | 是 | 用户问题 |
| documents | []string | 是 | 候选文档列表 |

接口响应：
```json
{
  "content": [
    {
      "index": 2,
      "id": 3757629,
      "score": 0.95
    },
    {
      "index": 0,
      "id": 3757630,
      "score": 0.82
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| content | []object | 排序后的文档列表 |
| content[].index | int | 原始文档在输入列表中的下标 |
| content[].id | int | 文档 ID（article_entries 表主键） |
| content[].score | float | 与用户问题的相关性得分（0~1） |

接口说明：
1. 使用 LLM 对候选文档进行语义重排序
2. 结果按 score 降序排列

---

### 4. 查询改写（Query Rewrite）

请求方式：POST
接口路由：`http://localhost:8086/v2/rewrite`

接口参数：
```json
{
  "history": ["用户：上一个问题", "助手：上一个回答"],
  "query": "那黄金呢？"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| history | []string | 否 | 历史对话记录 |
| query | string | 是 | 当前用户问题 |

接口响应：
```json
"黄金的最新走势和分析是怎样的？"
```

接口说明：
1. 将含指代、省略的追问改写为独立可检索的完整问题
2. 当对话消息超过 19 条时，后端会自动调用此接口进行查询改写

### 5. 对话列表

请求方式：GET
接口路由：`http://localhost:8086/v2/session/list`

接口参数（Query Parameters）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cursor | int64 | 否 | 分页游标，从 0 开始 |
| limit | int64 | 否 | 每页数量，默认 10，最大 100 |

请求示例：`GET http://localhost:8086/v2/session/list?cursor=0&limit=20`

接口响应：
```json
{
  "sessions": [
    "chatHistory:0f540b68-7186-4c24-b0ab-f43a8174eb33",
    "chatHistory:6c700834-3c8b-481e-a8d5-aeada9ef552c"
  ],
  "next_cursor": 0
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| sessions | []string | 会话 key 列表，格式为 `chatHistory:<session_id>` |
| next_cursor | int64 | 下一页游标，为 0 表示已无更多数据 |

接口说明：
1. 使用 Redis SCAN 遍历，分页游标从 0 开始
2. `next_cursor` 为 0 时表示遍历完毕

---

### 6. 对话详情

请求方式：GET
接口路由：`http://localhost:8086/v2/session/history`

接口参数（Query Parameters）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| session_id | string | 是 | 会话 ID（不含 `chatHistory:` 前缀） |

请求示例：`GET http://localhost:8086/v2/session/history?session_id=4f1b8f26-2b0a-4465-8058-d07c43882bcc`

接口响应：
```json
[
  {
    "role": "user",
    "content": "说一下白银的蛙跳与逼仓相关的内容"
  },
  {
    "role": "assistant",
    "content": "## 一、概念解释\n\n白银的蛙跳是指..."
  }
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| role | string | 角色：`user` 或 `assistant` |
| content | string | 消息内容 |

接口说明：
1. 返回最近 20 条消息记录
2. 消息按时间顺序排列
